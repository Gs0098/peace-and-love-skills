## 一句话说明

v15 是一套**将 AI Agent 的 SKILL.md 冗余内容安全去除**的规则体系。它不依赖具体业务，而是提供一套通用判断标准：**什么能删、什么不能删、删了怎么验证**。

> 用最少的字，传递最多的判断力。

---

## 快速开始

### 使用方式

**手动精简（推荐先看一遍规则）：**

1. 读取你的 SKILL.md
2. 对每一段内容执行三层检查：
   - 已最精炼？是 → 保留，跳到下一段
   - 少了会错？是 → 保留，跳到下一段
   - 删了会变？是 → 保留，跳到下一段
   - 全否 → 标记为可删除
3. 锚点保护：表格/公式/清单/脚本/阈值 **禁止删除**
4. 删除标记内容
5. 用测试数据跑 AB 版，确认 B 版无退化

**完整规则文档：** [outputs/skills-slimming-v15.md](outputs/skills-slimming-v15.md)

## 核心原理

### 三层判断模型

```
           ┌─ 是 → 保留
内容进来 → 已最精炼？
           └─ 否 ↓
           ┌─ 是 → 保留
           少了会错？
           └─ 否 ↓
           ┌─ 是 → 保留
           删了会变？
           └─ 否 → 删除 ✓
```

### 锚点 vs 覆层

| 类型     | 定义                       | 策略         |
| -------- | -------------------------- | ------------ |
| **锚点** | Agent 做决策必须依赖的信息 | 绝对禁止删除 |
| **覆层** | 装饰性/重复性/背景性内容   | 可以安全删除 |

**锚点清单（禁止删除）：**
- 表格中的任何行列
- 公式、阈值、维度
- 检查清单、验证步骤
- 脚本路径、命令定义
- 个性化变量 `{{xxx}}`

**覆层清单（可以直接删）：**
- "You are an expert..." 角色声明
- "Let's get started!" 激励性语言
- 目录/导航块（TOC）
- 同一约束第 2+ 次出现
- 无实质内容的外部引用

### 压缩空间检测

不是所有内容都值得压缩。每字承载独立语义的段落，判定为"已最优"，不应该强行压缩。例如 31 行的 changelog Skill，v15 正确识别为 0 压缩空间，不做任何删除。

### 按压缩率分布

| 压缩率区间   | Skill 数 | 代表                                       |

### 压缩率 ≠ 质量损失

| 压缩率   | Skill                  | AB 差异  |
| -------- | ---------------------- | -------- |
| **-67%** | content-strategist     | **0 分** |

> 质量由"保留了什么锚点"决定，而非"减了多少字"。

## 验证方法

每个 Skill 独立执行 AB 测试：

| 版本        | 输入                         | 输出     |
| ----------- | ---------------------------- | -------- |
| A（原版）   | 原始 SKILL.md + 测试数据     | OUTPUT-A |
| B（精简版） | v15 精简 SKILL.md + 测试数据 | OUTPUT-B |

### 评测维度（6 维 120 分）

| 维度         | 满分 |
| ------------ | ---- |
| 结构完整性   | 20   |
| 内容准确性   | 30   |
| 可操作性     | 20   |
| 无冗余       | 15   |
| 格式一致性   | 15   |
| 业务流程覆盖 | 20   |

### 退化判定

任一条件触发即判定退化：
- 内容准确性 < 15/30
- 业务流程覆盖 < 10/20
- B 低于 A 超 15 分
- 结构完整性 < 10/20

---

## 高维度发现

通过 100 个 Skill 验证，我们沉淀了 5 条通用规律：

1. **压缩率与质量无线性关系** — 锚点决定质量，不是减了多少
2. **模板驱动型 Skill 免疫压缩** — 只要模板和约束保留，可砍 67% 无损
3. **上下文块可安全移除** — 步骤已自文档化，Agent 靠锚点决策
4. **命令-引用型天然紧凑** — 锚点密度接近 100%，已最优可识别
5. **锚点密度决定压缩上限** — 可预测一个 Skill 的理论压缩空间
